Create scripts/rollback.sh that reads the active slot, computes the other slot, and switches Nginx to it.

**Requirements:**
- Use `set -euo pipefail`
- SSH key default: `${SSH_KEY:-$HOME/.ssh/ec2-static-site-key.pem}`
- Host default: `${HOST:-13.236.205.122}`
- SSH opts: `-o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null`
- No arguments required — rollback always flips to the OTHER slot

**Logic:**
1. Read current active slot via SSH: `cat /var/run/blue-green-state 2>/dev/null || echo blue`
2. Compute rollback target:
   ```bash
   if [[ "$active_slot" == "blue" ]]; then
       rollback_slot="green"
   else
       rollback_slot="blue"
   fi
   ```
3. Print: `=== Rolling back: $active_slot -> $rollback_slot ===`
4. Execute rollback on remote host (heredoc):
   - `sudo ln -sf /etc/nginx/sites-available/blue-green-${rollback_slot}.conf /etc/nginx/sites-enabled/blue-green`
   - `sudo nginx -t`
   - `sudo nginx -s reload`
   - `echo '$rollback_slot' | sudo tee /var/run/blue-green-state > /dev/null`
   - Verify: `echo "Active slot: $(cat /var/run/blue-green-state)"`
5. Print: `SUCCESS: Rolled back to $rollback_slot`
6. Print: `Previous slot ($active_slot) is still running at port <old_port>`

**Port computation for the "still running" message:**
- `[[ "$active_slot" == "blue" ]] && old_port=3001 || old_port=3002`
- `[[ "$rollback_slot" == "blue" ]] && new_port=3001 || new_port=3002`

**Important:**
- The heredoc uses single-quoted delimiter `'REMOTE'` so variables expand on remote
- State file write must happen AFTER nginx -s reload succeeds
- Both old and new slots' containers remain running — no docker compose commands

**Usage:** `./rollback.sh` (no arguments needed)
**Exit code:** 0 on success, non-zero on any step failure

Create scripts/logs.sh that retrieves container logs and Nginx logs via SSH.

**Requirements:**
- Use `set -euo pipefail`
- SSH key default: `${SSH_KEY:-$HOME/.ssh/ec2-static-site-key.pem}`
- Host default: `${HOST:-13.236.205.122}`
- SSH opts: `-o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null`

**Arguments:**
- `$1` = target: `blue`, `green`, `nginx`, or `all` (default: `all`)
- `--lines N` or `-n N`: number of lines to tail (default: 100 for containers, 50 for nginx)
- `-f` or `--follow`: follow mode (like `tail -f`) — requires keeping SSH connection open

**Container log retrieval (blue or green):**
- SSH into EC2 and run: `docker compose -f /opt/<slot>/docker-compose.yml logs --tail=<N>`
- Example: `docker compose -f /opt/blue/docker-compose.yml logs --tail=100`

**Nginx log retrieval:**
- Access log: `sudo tail -n <N> /var/log/nginx/access.log` (or `tail -n <N` for local access)
- Error log: `sudo tail -n <N> /var/log/nginx/error.log`
- Or combined: `sudo tail -n <N> /var/log/nginx/access.log /var/log/nginx/error.log`

**Argument parsing:**
```bash
target="all"
lines=100
follow=false

while [[ $# -gt 0 ]]; do
    case "$1" in
        blue|green|nginx|all) target="$1"; shift ;;
        -f|--follow) follow=true; shift ;;
        -n|--lines)
            lines="${2:-100}"; shift 2 ;;
        *) echo "Usage: $0 [blue|green|nginx|all] [-f] [-n lines]"; exit 1 ;;
    esac
done
```

**Output sections:**
- For `blue`: echo `=== Blue Container Logs (last $lines lines) ===` then logs
- For `green`: echo `=== Green Container Logs (last $lines lines) ===` then logs
- For `nginx`: echo `=== Nginx Access Logs (last $lines lines) ===` then access log; echo `=== Nginx Error Logs (last $lines lines) ===` then error log
- For `all`: output all four sections

**Follow mode:**
- For container logs with -f: `docker compose -f /opt/<slot>/docker-compose.yml logs -f`
- For nginx with -f: `sudo tail -f /var/log/nginx/access.log /var/log/nginx/error.log`
- In follow mode, the script keeps the SSH connection open and streams output

**Usage examples:**
- `./logs.sh blue` — last 100 lines of blue container logs
- `./logs.sh green -n 500` — last 500 lines of green container logs
- `./logs.sh nginx` — last 50 lines of nginx access and error logs
- `./logs.sh all -n 200` — all logs, last 200 lines each
- `./logs.sh nginx -f` — follow nginx logs in real time
- `./logs.sh blue -f` — follow blue container logs in real time
</action>
