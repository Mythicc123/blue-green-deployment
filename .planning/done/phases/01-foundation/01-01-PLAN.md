<task type="auto">
  <name>Task 2: Enhance /health endpoint to verify MongoDB connectivity</name>
  <files>app/src/index.js</files>
  <action>
Modify app/src/index.js /health endpoint to verify MongoDB connectivity, not just return static OK.

Replace the current health check:
```javascript
app.get('/health', (req, res) => {
  res.json({ status: 'ok' });
});
```

With a MongoDB-pinging version that checks the mongoose connection is active:
```javascript
app.get('/health', async (req, res) => {
  try {
    if (mongoose.connection.readyState !== 1) {
      return res.status(503).json({ status: 'error', message: 'MongoDB not connected' });
    }
    await mongoose.connection.db.adminCommand({ ping: 1 });
    res.json({ status: 'ok', mongo: 'connected' });
  } catch (err) {
    res.status(503).json({ status: 'error', message: 'MongoDB ping failed' });
  }
});
```

This ensures the Docker Compose healthcheck (which curls /health) verifies full stack health including MongoDB connectivity. The `readyState === 1` check confirms the connection is open (0=disconnected, 1=connected, 2=connecting, 3=disconnecting).
</action>
