**Requirements:**

- SSH key default: `${SSH_KEY:-$HOME/.ssh/ec2-static-site-key.pem}`
- Host default: `${HOST:-13.236.205.122}`

1. Change to slot directory: `cd /opt/$inactive_slot`
2. Read DOCKER_IMAGE from .env: `DOCKER_IMAGE=$(grep DOCKER_IMAGE .env | cut -d= -f2)`

- After deploy, echo success message including which slot was deployed to
- Exit with non-zero code if docker compose pull or docker compose up fails

**Example heredoc pattern:**
```bash
ssh $SSH_OPTS -i "$SSH_KEY" ubuntu@$HOST bash << 'REMOTE'
    set -euo pipefail
    cd /opt/$inactive_slot
    DOCKER_IMAGE=\$(grep DOCKER_IMAGE .env | cut -d= -f2)
    echo "Pulling image: $DOCKER_IMAGE"
    docker compose pull
    docker compose up -d
    echo "Deployed to $inactive_slot"
REMOTE
```

Create scripts/health-check.sh that polls the health endpoint of a given slot until it returns healthy or times out.

**Arguments:**
- `$1` = slot name (`blue` or `green`) — REQUIRED
- `$2` = timeout in seconds (default: 60)
- `$3` = poll interval in seconds (default: 3)

**Validation:**
- If `$1` is empty or not `blue`/`green`, print usage and exit 1

**Port mapping:**
```bash
case "$slot" in
    blue)  port=3001 ;;
    green) port=3002 ;;
    *)     echo "Error: slot must be blue or green" >&2; exit 1 ;;
esac
```

**Health check logic (loop):**
- URL: `http://localhost:${port}/health`
- Run curl on REMOTE host via SSH: `ssh $SSH_OPTS -i "$SSH_KEY" ubuntu@$HOST "curl -sf '$url'" 2>/dev/null`
- Validate response by grepping for `"status":"ok"`: `echo "$response" | grep -q '"status":"ok"'`
- If healthy: echo `"PASS: Health check passed after ${waited}s"`, echo the full response, exit 0
- If not healthy: echo `"  [${waited}s] Not ready yet..."`, sleep interval, increment waited
- If timeout: echo `"FAIL: Health check timed out after ${timeout}s"`, exit 1

**Variable initialization:**
- `waited=0`
- `interval="${3:-3}"` (use third arg or default to 3)
- `timeout="${2:-60}"` (use second arg or default to 60)

**Example output on success:**
```
Polling http://localhost:3002/health (timeout: 60s, interval: 3s)...
  [0s] Not ready yet...
  [3s] Not ready yet...
  [6s] Not ready yet...
PASS: Health check passed after 9s
{"status":"ok","mongo":"connected"}
```

**Example output on timeout:**
```

  [0s] Not ready yet...
FAIL: Health check timed out after 60s
```

Create scripts/switch-nginx.sh that switches the Nginx symlink to the target slot and updates the state file.

**IMPORTANT:** This is a NEW script. Do NOT modify scripts/switch.sh (Phase 1). This script adds state file updates to the existing switch pattern.

**Arguments:**
- `$1` = target slot (`blue` or `green`) — REQUIRED

**Validation:**
- If `$1` is empty or not `blue`/`green`, print usage and exit 1

**Steps (all via single SSH heredoc to the remote host):**
1. Validate target config exists: `test -f /etc/nginx/sites-available/blue-green-${target}.conf`
2. Switch symlink: `sudo ln -sf /etc/nginx/sites-available/blue-green-${target}.conf /etc/nginx/sites-enabled/blue-green`
3. Validate Nginx config: `sudo nginx -t`
4. Reload Nginx: `sudo nginx -s reload`
5. Update state file: `echo '$target' | sudo tee /var/run/blue-green-state > /dev/null`
6. Verify symlink: `readlink /etc/nginx/sites-enabled/blue-green`
7. Verify state file: `cat /var/run/blue-green-state`

**Print status messages (locally) before each step:**
- `[1/5] Switching symlink to blue-green-${target}.conf...`
- `[2/5] Validating Nginx configuration...`
- `[3/5] Reloading Nginx...`
- `[4/5] Updating state file...`
- `[5/5] Verifying...`

**On success:** echo `SUCCESS: Switched to $target`
**On failure:** Each step exits non-zero on failure due to `set -euo pipefail` inside heredoc

**State file write note:** The heredoc uses single-quoted delimiter `'REMOTE'` so `$target` is expanded on the remote side, not locally. The `tee` command requires sudo since /var/run/ requires root.

**The state file write must happen AFTER nginx -s reload succeeds (not before).** This ensures state file and Nginx routing stay in sync.
</action>
